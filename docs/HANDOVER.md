# Handover & Next Steps

통합 교육 학습 놀이터로의 아키텍처 개편 및 구구단 15단 챌린지 탑재가 성공적으로 완료되었습니다. 다음 작업자 혹은 다음 세션을 위해 상세 내역을 인계합니다.

---

## 🎯 현재 상태

- **메인 런처**: 받아쓰기와 구구단을 아름다운 Glassmorphism 카드로 골라 진입할 수 있는 최상위 관문 구현 완료.
- **받아쓰기 앱**: 1~2학년 데이터셋 유지 및 정상 작동.
- **구구단 15단 앱**: 2~15단 다중 선택, 주관식/객관식 모드 선택, 오답 보기 정밀 알고리즘 작동, 가상 키패드 제어, 오답 분석 리포트 구현 완료.
- **빌드 상태**: Vite 프로덕션 빌드 최적 상태 유지 (`npm run build` 검증 완료).

## 💡 오늘 배운 핵심 개념 요약 (TL;DR)

1. **상태 분산 및 격리 (State Decoupling)**:
   - 복잡도가 서로 다른 두 도메인을 개발할 때는 결합도를 차단하기 위해 폴더 구조를 격리하고 상위 라우터(`App.jsx`)에서는 단순 모드 분기 및 홈 이동 핸들러만 제공하여 응집도를 높여야 함.
2. **인지 모델링 기반 오답(Distractor) 설계**:
   - 교육용 앱에서는 무작위 난수가 아닌 학습자의 인지적 실수의 규칙(인접 곱셈, 자릿수 표기 오류)을 모델링하여 오답을 수학적으로 생성하는 것이 학습 만족도와 훈련 강도를 비약적으로 높임.
3. **가상 키패드와 리액트 라이프사이클**:
   - 가상 키패드의 동적 클릭과 실시간 키보드 `keydown` 이벤트를 매끄럽게 연결하기 위해 상태 종속성 관리를 정밀히 처리해야 함. 또한, 보기가 렌더링마다 섞이는 것을 차단하기 위해 문제 이동 타이밍에 맞춰 보기를 로컬에 캐싱하는 라이프사이클 관리가 필수적임.

## ⚠️ 실패 이력 (지뢰밭)

- **[시도했던 오답 보기 실시간 렌더링 방식]**:
  - *증상*: 객관식 모드에서 사용자가 키보드를 누르거나 타이핑할 때마다 보기가 실시간으로 재셔플되며 순서가 휙휙 바뀌는 치명적인 UI 레이아웃 흔들림 현상 발생.
  - *원인*: 컴포넌트 렌더링 본문 내에서 `generateMultipleChoices`를 직접 호출하여 리턴값을 JSX에 바인딩했기 때문. 리액트 상태가 변해 리렌더링이 일어날 때마다 셔플 함수가 새로 작동함.
  - *교훈/해결*: 문제 인덱스(`currentIndex`)가 전이되는 시점에 맞추어 `useEffect` 내부에서 보기를 1회만 계산하고, 이를 `currentChoices` 상태 변수에 할당해 사용하는 방식으로 우회하여 완벽하게 화면 요동 현상 방어.
